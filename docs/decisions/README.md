# Architecture Decision Records (ADRs)

Capture significant technical and product decisions here using the ADR template below. Number files sequentially (`ADR-0001.md`, `ADR-0002.md`, …) and reference them from the master PRD or changelog when relevant.

## ADR Template
```
# ADR-XXXX – Title
- **Date:** YYYY-MM-DD
- **Status:** Proposed | Accepted | Superseded | Deprecated
- **Context:** Background, constraints, and problem statement.
- **Decision:** The chosen solution.
- **Alternatives Considered:** Options evaluated and why they were rejected.
- **Consequences:** Positive/negative outcomes, follow-up actions.
- **Related Documents:** Links to PRDs, tickets, pull requests.
```
