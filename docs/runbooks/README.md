# Operational Runbooks

Document repeatable procedures for operating the Illajwala VisitNow platform. Each runbook should be a standalone markdown file named after the scenario (e.g., `incident-response.md`, `release-train.md`, `oncall-handoff.md`).

## Minimum Contents Per Runbook
- **Purpose:** Why the runbook exists.
- **Owners:** Teams/roles responsible.
- **Pre-requisites:** Access rights, tools, credentials.
- **Procedure:** Step-by-step instructions with validation checkpoints.
- **Fallbacks:** Contingency plans if a step fails.
- **Communication:** Notification templates, escalation paths.
- **Post-Completion:** Verification steps, logging, documentation updates.

## Suggested Runbooks to Author
- Incident response workflow.
- Release management & hotfix protocol.
- Support escalation ladder.
- On-call rotation handoff.
- Disaster recovery & backup restore test.
